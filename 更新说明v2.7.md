# v2.7 调度 outage 判断更新说明

## 1. 本次修改

v2.7 修正了 Sionna ILLA 在极低 SINR 下返回最低可用 MCS 时，调度器仍把该 MCS 当作可用链路并计算正速率的问题。

Sionna ILLA 会先计算所有可用 MCS 的 TBLER，并选择满足目标 BLER 的最高 MCS：

$$
m^\star=\max\{m:\operatorname{TBLER}(\gamma,m)\leq P_{\mathrm{target}}\}.
$$

当集合非空时，返回最高的可行 MCS；当集合为空时，Sionna 仍会返回最低可用 MCS。在当前 PDSCH 表配置中，该最低可用 MCS 通常为 MCS 3。因此，只保存 `su_mcs=3` 无法区分以下两种情况：

- MCS 3 的 TBLER 不超过目标值，链路可用；
- 所有 MCS 都不满足目标 BLER，ILLA 只是按接口约定回退到 MCS 3，链路实际处于调度 outage。

v2.7 增加独立的 outage 判断：

$$
P_{\min}=\operatorname{TBLER}(\gamma,m_{\min}).
$$

若：

$$
P_{\min}\leq P_{\mathrm{target}},
$$

则最低 MCS 可用；若：

$$
P_{\min}>P_{\mathrm{target}},
$$

则候选链路处于调度 outage，并设置：

$$
R_{\mathrm{pred}}=0.
$$

默认目标仍由 `system.target_bler` 控制，当前默认值为 `0.1`。

## 2. 代码改动

### 2.1 链路适配器

`beam_sls/link_adaptation.py` 的基础链路适配器新增：

- `is_outage_from_sinr_db()`；
- `is_outage_from_sinr_lin()`。

这两个接口会在 MCS 选择完成后，用同一 SINR、同一 MCS 和同一 TBLER 后端再次检查目标 BLER。非有限、非法或高于目标值的 TBLER 都会被判定为 outage。

因此，Sionna SYS、fallback precomputed table 和 surrogate 路径都通过同一个上层接口暴露 outage 状态。

### 2.2 SU 测量与反馈

`beam_sls/measurement.py` 的 `MeasurementResult` 新增：

```text
su_outage[ue, beam]
```

每个候选服务波束现在同时保存：

- `su_snr_db`；
- `su_mcs`；
- `su_outage`。

`beam_sls/feedback.py` 的 `ServiceCandidate` 也新增 `su_outage` 字段，并写入 UE report。这样，`su_mcs=3` 和“该候选是否满足目标 BLER”不再混为一个状态。

### 2.3 Limited-feedback 调度路径

baseline、Top-K conflict ID 和 threshold conflict set 使用上报候选中的 `su_outage`：

- `su_outage=false`：按 `su_mcs` 计算预测速率；
- `su_outage=true`：保留 `su_mcs` 作为诊断信息，但令 `predicted_rate_mbps=0`。

这里不会把 outage 人为编码成 `MCS=0`。MCS 字段仍表示 ILLA 的实际返回值，outage 由独立布尔字段表示。

### 2.4 Full Gamma 调度路径

Full Gamma 的预测 SINR 会随共同调度用户组合变化，因此不能直接复用 SU outage。

每次组合评估现在执行：

1. 根据完整 Gamma 重建共同调度干扰；
2. 计算预测 MU-SINR；
3. 由 ILLA 选择预测 MCS；
4. 用该预测 SINR 和预测 MCS 查询 TBLER；
5. 若 TBLER 高于目标值，则标记 `predicted_outage=true`，并令预测速率为 0。

因此，Full Gamma 与 SU 路径采用相同的目标 BLER 判据，但使用各自对应的预测 SINR。

### 2.5 各调度算法的行为

- `greedy`：outage 候选的增量速率为 0，不会作为正收益候选加入调度结果；
- `exhaustive`：outage 候选的 standalone upper bound 为 0，可被 zero-upper-bound pruning 去除；
- `hard_conflict_greedy`：SU outage 候选不会进入初始候选池；
- 分站点和分扇区调度会聚合各 domain 的 outage 统计。

## 4. 配置与运行方式

本次更新不需要新增 YAML 参数。现有配置可以直接运行，例如：

```powershell
python -m beam_sls.run `
  --config configs/v2_one_site_three_sector_sionna_strict.yaml `
  --out runs/v2_7_outage_check
```

目标 BLER 继续使用：

```yaml
system:
  target_bler: 0.1
```

链路抽象继续使用原有配置，例如严格 Sionna SYS 模式：

```yaml
link_abstraction:
  mode: sionna_sys_precomputed_bler
  mcs_table_index: 1
  mcs_category: 1
```

不需要手动配置最低 MCS。适配器使用 ILLA 返回的表内最低可用 MCS，并用 PHY abstraction 的 TBLER 结果判断是否满足目标。

## 6. 结果解释

以下记录是合法且预期的：

```text
su_mcs = 3
su_outage = true
predicted_rate_mbps = 0
```

它表示 ILLA 因没有任何 MCS 满足目标 BLER，按接口约定返回了最低可用 MCS 3；调度器已正确识别该链路不可用，没有把 MCS 3 对应的名义 TBS 当作可实现速率。

相反：

```text
su_mcs = 3
su_outage = false
predicted_rate_mbps > 0
```

表示 MCS 3 在当前 SINR 下确实满足目标 BLER，可以参与调度。

本次改动只修正调度预测与候选选择，不改变 ILLA 返回的 MCS 编号，也不修改真实链路上的 TBLER、ACK/NACK、OLLA 或 goodput 抽样规则。

## 7. 验证覆盖

新增回归验证覆盖：

- 低 SINR 时仍返回 MCS 3，但 `su_outage=true`；
- SU outage 候选的预测速率为 0，且不会压过正常候选被调度；
- Full Gamma 使用预测 MU-SINR 重新执行 outage 检查；
- `ScheduledLink`、候选报告和调度统计正确写出 outage 字段；
- Sionna 低 SINR、非法低 MCS TBLER 场景被判定为 outage。

## 8. 特定距离范围垂直波束扫描

### 8.1 功能说明

v2.7 增加 `distance_range` 垂直波束模式。该模式根据 UE 水平距离范围和 BS/UE 高差计算几何下倾角范围，再在阵列相位域均匀采样，最后直接生成连续相位的垂直码本。它不会把目标角度取整到已有垂直 DFT index，因此可以准确覆盖指定距离区间。

设：

- 最近和最远水平距离为 $r_{\min}$、$r_{\max}$；
- BS 与 UE 高差为 $\Delta h=h_{\mathrm{BS}}-h_{\mathrm{UE}}>0$；
- 向上为正仰角，几何下倾角 $\alpha>0$；
- 垂直阵元间距为 $d_v$，单位为波长。

则：

$$
\alpha(r)=\arctan\left(\frac{\Delta h}{r}\right),\qquad
\theta(r)=-\alpha(r)<0.
$$

这里 $\theta<0$ 是代码内部的 elevation，表示波束向下。相邻垂直阵元的相位步进为：

$$
\phi(r)=2\pi d_v\sin\theta(r)<0.
$$

对 $M_v$ 个垂直波束，在近端和远端相位之间均匀采样：

$$
\phi_m=\phi(r_{\min})+
\frac{m}{M_v-1}\left[\phi(r_{\max})-\phi(r_{\min})\right],
\quad m=0,\ldots,M_v-1.
$$

第 $m$ 个垂直码字为：

$$
\mathbf w_{v,m}=\frac{1}{\sqrt{N_v}}
\left[1,e^{j\phi_m},\ldots,e^{j(N_v-1)\phi_m}\right]^T.
$$

负的 $\phi_m$ 是本项目坐标约定下保证波束下倾的关键。代码没有使用相反符号；回归测试会扫描 $-60^\circ$ 到 $20^\circ$ 的阵列响应，并确认每个主瓣峰值均位于目标负仰角。

### 8.2 代码改动

- `beam_sls/codebook.py`
  - `ArrayConfig` 新增距离范围和高度差参数；
  - 新增距离到几何角、负垂直相位以及相位域均匀采样；
  - 现有水平 DFT 码本保持不变，只将垂直向量替换为连续相位码字；
  - joint、per-panel 和 panel-polarization-subarray 三条 TX 码本路径均支持该模式；
  - `BeamId` 新增 `vertical_phase_rad`、`elevation_deg`、`downtilt_deg`、`horizontal_distance_m`。
- `beam_sls/config.py`
  - 距离模式未显式填写范围时，继承 `scenario.min_ue_distance_m/max_ue_distance_m`；
  - 未显式填写高度时，继承 `topology.bs_height_m/ue_height_m`。
- `beam_sls/topology.py`、`beam_sls/coverage.py`
  - UE 高度改为读取 `topology.ue_height_m`，默认仍为 1.5 m。
- `beam_sls/channel.py`
  - NumPy fallback 几何信道的 TX elevation 改为真实 BS→UE 几何负仰角；此前它错误地在 0° 附近生成垂直角，无法正确验证下倾覆盖。
- `tests/test_array_config.py`
  - 新增相位均匀性、距离端点、主瓣下倾方向、默认 RF 路径、BeamId 元数据和配置继承回归测试。

旧配置的默认 `vertical_beam_mode: scan`、DFT 扫描和 `fixed_v_index` 行为保持不变。

### 8.3 配置方法

最小配置如下。距离、BS 高度和 UE 高度直接继承现有场景参数：

```yaml
scenario:
  min_ue_distance_m: 35.0
  max_ue_distance_m: 250.0

topology:
  bs_height_m: 25.0
  ue_height_m: 1.5

tx_array:
  vertical_beam_mode: distance_range
  num_beams_h: 4
  num_beams_v: 4
  max_beams: 16
```

也可以在 `tx_array.vertical_beam` 中单独覆盖距离和高差：

```yaml
tx_array:
  vertical_beam_mode: distance_range
  num_beams_h: 4
  num_beams_v: 4
  max_beams: 16
  vertical_beam:
    min_horizontal_distance_m: 35.0
    max_horizontal_distance_m: 250.0
    height_difference_m: 23.5
```

`height_difference_m` 的优先级高于 `bs_height_m-ue_height_m`。距离模式不能与 `fixed_v_index` 同时使用；若同时配置，程序会直接报错，避免静默生成错误方向的波束。

运行方式不变：

```powershell
python -m beam_sls.run `
  --config configs/your_distance_range_config.yaml `
  --out runs/v2_7_distance_vertical_scan
```

### 8.4 输出与方向检查

生成后的 `metrics/beams.csv` 新增：

| 字段 | 含义 | 下倾波束的预期符号 |
|---|---|---|
| `vertical_phase_rad` | 相邻垂直阵元相位步进 | 小于 0 |
| `elevation_deg` | 代码内部仰角，向上为正 | 小于 0 |
| `downtilt_deg` | 正值几何下倾角 | 大于 0 |
| `horizontal_distance_m` | 该相位点反算的波束中心水平距离 | 位于配置范围内 |

距离模式使用连续相位，因此 `v_index` 留空是预期行为，不表示码本生成失败。

可以直接检查：

```powershell

  Select-Object cell,panel,h_index,v_index,vertical_phase_rad,elevation_deg,downtilt_deg,horizontal_distance_m |
  Sort-Object cell,panel,h_index,horizontal_distance_m
```
